Приложение G — Решатель LPSolve

Пример G.1 Рассмотрим задачу

\[ \begin{aligned} &\mathrm{Z}=11x_1+5x_2+4x_3 \rightarrow \max\\ &\begin{cases} 3x_1+2x_2+8x_3 \leq 11\\ 2x_1+x_3 \leq 5 \\ 3x_1+3x_2+x_3 \leq13 \\ x_1,x_2,x_3 \geq 0 \end{cases} \end{aligned} \] В данном случае решается задача макисимизации целевой функции \(\mathrm{Z}=11x_1+5x_2+4x_3\). При ограничениях вида \(\mathbf{Ax} \leq \mathbf{b}\), где

\[ \mathbf{A}= \begin{pmatrix} 3 & 2 & 8 \\ 2 & 0 & 1 \\ 3 & 3 & 1 \end{pmatrix} \] матрица коэффициентов ограничений (матрица ограничений). \(\mathbf{b}=(11,5,13)\) вектор-столбец правых частей.

G.1 lpSolve

G.1.1 Установка

Важное уведомление

Убедитесь в том, что Rtools установлены в системе Глава 1.2

```{r}
install.packages('lpSolve')
```

G.1.2 Модель

На основе примера Пример G.1. Для начала требуется определить целевую функцию \(\mathrm{Z}\). В пакете lpsolve целевая функция задается как вектор Глава 6.3 коэффициентов.

Z <- c(11, 5, 4)

Ограничения задаются в виде матрицы коэффициентов \(\mathbf{A}\). Пакет может работать с общей формулировкой задачи Глава 1 поэтому приведение к стандартной форме не требуется

A <- rbind(c(3, 2, 8),
           c(2, 0, 1),
           c(3, 3, 1))
Совет

Если требуется задать ограничения вида \(x_i\leq b_i, \quad \forall x_i\) то рекомендуется задать коэффициенты матрицы \(\mathbf{A}\) ограничений (единичную матрицу Определение 7.6) с помощью функции функцию base::diag(). Например

diag(3)
     [,1] [,2] [,3]
[1,]    1    0    0
[2,]    0    1    0
[3,]    0    0    1

Правые части и знаки ограничений задаются отдельно

B <- c(11, 5, 13)
S <- c('<=', '<=', '<=')
Совет

Если требуется задать много однотипных ограничен рекомендуемый применить функцию base::rep() . Например {r} B <- rep('<=', 10).

G.1.3 Решение

Для поиска решения вызывается функция lpSolve::lp()

library(lpSolve)

optimum <- lp(           
1  direction = 'max',
2  objective.in = Z,
3  const.mat = A,
4  const.dir = S,
5  const.rhs = B,
6  compute.sens = TRUE)
optimum

2
Задать целевую (objective) функцию вектором коэффициентов.
3
Задать матрицу коэффициентов ограничений.

Максимальное значение целевой функции получается

optimum$objval
[1] 36.25

Это значение достигается при следующих значения переменных. Его можно получить из объекта решения optimum по ключу solution

optimum$solution
[1] 2.50 1.75 0.00
Совет

Если в задаче много переменных, то иногда бывает трудно определить какие из них приняли ненулевые значения. Отображения результатов можно упростить используя функцию base::which()

Например

x <- c(0,1,2,0,4,5,0,6,8,0,0,1)
x
 [1] 0 1 2 0 4 5 0 6 8 0 0 1

Узнать индексы ненулевых элементов

ind <- which(x!=0)
ind
[1]  2  3  5  6  8  9 12

И вывести только ненулевые элементы

x[ind]
[1] 1 2 4 5 6 8 1

G.1.4 Анализ чувствительности оптимального решения

Целевой функции

Диапазон в кортом можно изменять коэффициенты целевой функции ?sec-linear-simplex-sensetivity-objective не меняя при этом характер решения (точку, соответствующую оптимальному решению) можно определить следующим образом

optimum$sens.coef.from
[1]  7.5e+00  0.0e+00 -1.0e+30
optimum$sens.coef.to
[1] 1.000000e+30 7.333333e+00 2.175000e+01

Соответственно точка \(( 2.5,1.75,0 )\) соответствующая оптимальному решению не сдвинется, если по одному изменять коэффициенты в следующих пределах

\[ \begin{aligned} &7.5 \leq c_1 \leq 10^{30}\\ &0 \leq c_2 \leq 7.3333333\\ &-10^{30} \leq c_3 \leq 21.75 \end{aligned} \]

Ограничений

Коэффициенты чувствительности по ограничениям ?sec-linear-simplex-sensetivity-constraint (также встречается название двойственные значения).

optimum$duals
[1]   2.50   1.75   0.00   0.00   0.00 -17.75

Это значение показывает как измениться целевая функция при изменении правой части ограничения на единицу.

Например значение целевой функции измениться с \(36.25\) на \(38.75\) (шаг \(2.5\)) при изменении ограничения \(3x_1+2x_2+8x_3 \leq 11\) на ограничение \(3x_1+2x_2+8x_3 \leq 12\).

Обратите внимание, количество двойственных значений гораздо больше, чем количество ограничений, которые были заданы в задаче изначально. Дело в том, что пакет неявным образом добавляет в модель ограничения вида \(x_i \geq 0 ,\forall x_i\), если другого не заданно. Это сделано для удобства разработчика.

Далее определяется диапазон в котором действительно данное двойственное значение.

optimum$sens.coef.from
[1]  7.5e+00  0.0e+00 -1.0e+30
optimum$sens.coef.from
[1]  7.5e+00  0.0e+00 -1.0e+30

Таким образом для первого ограничения \(3x_1+2x_2+8x_3 \leq 11\) двойственное значение \(2.5\) сохраняется при изменении правой части \(7.5 \leq b_1 \leq 7.5\)

G.1.5 Целочисленная модель

Для решения задачи Пример G.1 в целых числах при построении модели добавляется параметр all.int=TRUE. В этом случае анализ чувствительности Глава G.1.4 не применим и параметр compute.sens = TRUE будет проигнорирован

optimum <- lp(           
  direction = 'max',
  objective.in = Z,
  const.mat = A,
  const.dir = S,
  const.rhs = B,
  all.int = TRUE)
optimum
Success: the objective function is 32 
optimum$solution
[1] 2 2 0

Аналогично для решения задачи двоичного программирования устанавливается параметр all.bin=TRUE

optimum <- lp(           
  direction = 'max',
  objective.in = Z,
  const.mat = A,
  const.dir = S,
  const.rhs = B,
  all.bin = TRUE)
optimum
Success: the objective function is 16 
optimum$solution
[1] 1 1 0

Задача целочисленного и двоичного программирования в пакете lpSolve решается методом ветвей и границ Глава 11.1.

G.2 lpSolveAPI

G.2.1 Установка

Важное уведомление

Убедитесь в том, что Rtools установлены в системе Глава 1.2

```{r}
install.packages('lpSolveAPI')
```

G.2.2 Модель

На основе примера Пример G.1 создаём объект модели задачи линейного программирования lpSolveAPI::make.lp() с 3 переменными и 0 ограничений (будут добавлены позднее).

library(lpSolveAPI)
lprob <- make.lp(0, 3)

Далее выбирается тип оптимизации lpSolveAPI::lp.control(), в данном случае максимизация. Вывод функции подавляется с помощью base::invisible(), чтобы отменить печать информации на экран.

invisible(lp.control(lprob, sense = 'max'))

Далее задаются ограничения. Ограничения матрицы \(\mathbf{A}\) вводятся построчно lpSolveAPI::add.constraint(). По умолчанию на переменные накладывается условие \(\forall{i}: x_i\geq0\)

add.constraint(lprob, c(3, 2, 8), '<=', 11)
add.constraint(lprob, c(2, 0, 1), '<=', 5)
add.constraint(lprob, c(3, 3, 1), '<=', 13)

Задание целевой функции lpSolveAPI::set.objfn(). Объект модели можно конструировать в произвольном порядке

set.objfn(lprob, c(11, 5, 4))

Итоговое представление модели можно получить вызвав объект модели

lprob
Model name: 
            C1    C2    C3        
Maximize    11     5     4

R2           2     0     1  <=   5
R3           3     3     1  <=  13
Kind       Std   Std   Std        
Type      Real  Real  Real        
Upper      Inf   Inf   Inf        
Lower        0     0     0

G.2.3 Решение

Далее вызовом lpSolveAPI::solve() полученная модель решается. В случае успеха возвращается status_code == 0

status_code <- solve(lprob)
status_code
[1] 0

Наилучшее значение целевой функции lpSolveAPI::get.objective()

get.objective(lprob)
[1] 36.25

Значение переменных, при которых достигнуто максимальное значение целевой функции lpSolveAPI::get.variables()

get.variables(lprob)
[1] 2.50 1.75 0.00

Целевая функция

get.sensitivity.obj(lprob)

$objtill
[1] 1.000000e+30 7.333333e+00 2.175000e+01
